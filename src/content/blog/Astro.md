このポートフォリオサイトは、**Astro** というWebフレームワークを採用して構築しています。
現在、Web開発の主流である Next.js などの SPA（Single Page Application）フレームワークではなく、なぜあえて Astro を選定したのか。
その技術的な特徴とやメリットをまとめてみました。

### Astro とは：MPAへの回帰と進化

Astro（アストロ）は、**コンテンツ重視のWebサイト**に特化した現代的なWebフレームワークです。
構造としては古典的な MPA（Multi Page Application）のアプローチを採りつつ、開発体験は React などのモダンなコンポーネント指向を取り入れている点が特徴です。

ドキュメント、ブログ、ポートフォリオといった「読み物」が主体のサイトにおいて、現状では最適解の一つと言えます。

#### 1. デフォルトで JavaScript を排除（Zero JS）

React や Vue で構築された一般的な SPA は、画面を描画するためにブラウザ側で大きな JavaScript バンドルを読み込み、実行（Hydration）する必要があります。これは特にモバイル回線などにおいて、Time to Interactive（操作可能になるまでの時間）の遅延を招く要因となります。

対して Astro は、ビルド時に HTML を完全にレンダリングし、**不要な JavaScript をクライアントへ送信しません。**
ユーザーは「純粋な HTML/CSS」だけを受け取るため、ブラウザのメインスレッドをブロックせず、爆速なページ表示を実現できます。

#### 2. アイランドアーキテクチャ (Islands Architecture)

Astro の技術的な面白さは、**「必要な部分だけを動的にハイドレーションする」**という設計思想（Partial Hydration）にあります。これを「アイランドアーキテクチャ」と呼びます。例えば、このページの大半は静的な HTML ですが、「いいねボタン」のようなインタラクティブな機能が必要な場合、**そのコンポーネント（島＝Island）だけ** React や Vue を起動させることができます。

** client:load ** ディレクティブを付与したコンポーネントのみが、ブラウザ上で JavaScript として機能します。「静的なHTMLの大海原」の中に、「動的なコンポーネントの島」が浮かんでいるイメージです。これにより、高いパフォーマンスとリッチなUIを両立させています。

####3. "Bring Your Own Framework" (UIにとらわれない)
Astro は独自の .astro ファイルを使用しますが、その内部で React, Vue, Svelte といった既存のUIライブラリを自由に利用可能です。
私は Web 開発の学習として React (Next.js) にも関心がありますが、Astro であれば「ベースは軽量な Astro で組み、複雑な状態管理が必要な UI だけ React を使う」という適材適所の技術選定が可能です。 特定のフレームワークにロックインされない柔軟性も、採用の大きな決め手でした。

#### まとめ
このサイトにおける技術選定の理由は以下の通りです。

- Performance First: 不要な JS を排除し、Core Web Vitals のスコアを最大化できる。
- Developer Experience: JSX ライクな記法で直感的に記述でき、Markdown/MDX との親和性が極めて高い。
- Scalability: 必要に応じて React などの資産をシームレスに組み込める。

単に情報を載せるだけでなく、Web技術そのものの面白さを体現する場として、Astro を採用しています。
