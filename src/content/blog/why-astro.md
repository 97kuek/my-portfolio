今まではnoteでブログを書いていたんですが、情報系学生というのもあり、Webのことを学ぶためにもポートフォリオを作ろうと思いました。

このポートフォリオサイトは、Webフレームワークに **Astro** を採用して構築しています。現在、Webフロントエンド開発の主流は`Next.js`などの SPA (Single Page Application) フレームワークです。しかし、なぜあえて今 `Astro`を選定したのか。Astroの魅力と理由をまとめてみました。

## Astro とは？

Astro (アストロ) は、**コンテンツ重視のWebサイト**に特化したフレームワークです。
アーキテクチャとしては古典的な MPA（Multi Page Application）のアプローチを採りつつ、開発体験（DX）においては React などのモダンなコンポーネント指向を取り入れているのが最大の特徴です。

ドキュメント、ブログ、そして今回のポートフォリオのような**読み物が主体のサイトにおいて、現状では最適解の一つ**と言えます。

## 1. デフォルトでJavaScriptを排除

`React` や `Vue` で構築された一般的な SPA は、画面を描画するためにブラウザ側で大きな JavaScript バンドルを読み込み、実行（Hydration）する必要があります。これは特にモバイル回線などにおいて、TTI（Time to Interactive：操作可能になるまでの時間）の遅延を招く要因となります。

対して Astro は、ビルド時に HTML をサーバーサイドで完全にレンダリングし、**不要な JavaScript をクライアントへ送信しません。**
ユーザーは「純粋な HTML/CSS」だけを受け取るため、ブラウザのメインスレッドをブロックせず、爆速なページ表示を実現できます。

## 2. アイランドアーキテクチャ

Astro の技術的な面白さは、**「必要な部分だけを動的にハイドレーションする」**という設計思想（Partial Hydration）にあります。これを「アイランドアーキテクチャ」と呼びます。

例えば、このページの大半は静的な HTML ですが、「いいねボタン」のようなインタラクティブな機能が必要な場合、**そのコンポーネント（島＝Island）だけ** React や Vue を起動させることができます。

コード上では `client:load` や `client:visible` といったディレクティブを付与したコンポーネントのみが、ブラウザ上で JavaScript として機能します。「静的なHTMLの大海原」の中に、「動的なコンポーネントの島」が浮かんでいるイメージです。これにより、高いパフォーマンスとリッチなUIの両立が可能になります。

## 3. "Bring Your Own Framework" (UIにとらわれない柔軟性)

Astro は独自の `.astro` ファイルを使用しますが、その内部で React, Vue, Svelte といった既存のUIライブラリを自由に利用可能です（Bring Your Own Framework）。

私は Web 開発の学習として React (Next.js) にも関心がありますが、Astro であれば「ベースは軽量な Astro で組み、複雑な状態管理が必要な UI だけ React を使う」という、適材適所の技術選定が可能です。特定のフレームワークにロックインされない柔軟性も、採用の大きな決め手でした。

## Astroのセットアップ
Astro の開発体験 (DX) が優れている点は、環境構築の容易さと、直感的なファイル構造にあります。
ここでは、実際にプロジェクトを立ち上げ基本的な構造を理解するまでのステップを紹介します。

Node.js (v18.14.1以上) 環境下であれば、以下のコマンド1つでセットアップが完了します。

```bash
npm create astro@latest
```

コマンドを実行すると、ターミナル上で対話形式のウィザードが起動します。宇宙飛行士のキャラクターが案内してくれるので、以下の項目などを選択していきます。

1. **Where should we create your new project?:** プロジェクトのディレクトリ名
2. **How would you like to start your new project?:** テンプレート選択（ブログ用、空のプロジェクトなど）
3. **Install dependencies?:** 依存関係のインストール（Yes推奨）
4. **Do you plan to write TypeScript?:** TypeScript の設定（Yes推奨。厳格度も選べます）

すべて答え終わると自動的に環境が構築されます。設定ファイルを手動で書く必要はほとんどありません。

```bash
cd <プロジェクト名>
npm run dev
```

これでローカルサーバー(http://localhost:4321)が立ち上がります。ちなみにポート番号**4321**はLiftoff!へのカウントダウンを意味しているらしいです(笑)

## まとめ

このサイトにおいて Astro を採用した理由は、以下の3点に集約されます。

- **Performance First**: 不要な JS を排除し、Core Web Vitals のスコアを最大化できる。
- **Developer Experience**: JSX ライクな記法で直感的に記述でき、Markdown/MDX との親和性が極めて高い。
- **Scalability**: 必要に応じて React などの資産をシームレスに組み込める拡張性。

単に情報を載せるだけでなく、Web技術そのものの面白さを体現する場として、今後も運用を続けていきます。
