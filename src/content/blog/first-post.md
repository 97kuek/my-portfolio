このポートフォリオサイトは、**Astro** というWebフレームワークを使って構築しています。
なぜ Next.js などの有名フレームワークではなく Astro を選んだのか、その技術的な特徴とメリットをまとめてみます。

### Astro とは

Astro（アストロ）は、**「コンテンツ重視のWebサイト」**を作るために設計された最新のWebフレームワークです。
ドキュメントサイト、ブログ、ポートフォリオ、ECサイトなど、静的なコンテンツが主役のサイトにおいて圧倒的なパフォーマンスを発揮します。

#### 最大の特徴：デフォルトで JS を排除（Zero JS）

React や Vue で作られたサイト（SPA）は、画面を表示するために巨大な JavaScript をブラウザに読み込ませる必要があります。これはスマホなどの回線では表示速度の低下につながります。

一方、Astro はビルド時に**HTML を完全に生成し、不要な JavaScript を自動的に取り除きます。**
これにより、ユーザーは純粋な HTML/CSS だけを受け取るため、サイトの表示が爆速になります。

### アイランドアーキテクチャ (Islands Architecture)

Astro の面白さは、**「必要な部分だけ動的にする」**という考え方です。これを「アイランドアーキテクチャ」と呼びます。

例えば、このブログページ全体は静的な HTML ですが、もし「いいねボタン」や「検索バー」のようなインタラクティブな機能が必要な場合、**その部分（島＝Island）だけ** React や Vue を動かすことができます。

```astro
---
// ここはサーバーサイド（ビルド時）にのみ実行される
import Header from '../components/Header.astro';
import LikeButton from '../components/ReactLikeButton.jsx';
---

<Header />
<h1>私のブログ</h1>
<p>記事の本文です...</p>

<LikeButton client:load />
```
**client:load** というディレクティブ（指示）をつけたコンポーネントだけが、ブラウザ上で JavaScript として水分補給（Hydration）されます。 海（静的HTML）の中に、島（動的コンポーネント）が浮いているイメージです。

### React / Vue との共存
Astro は独自の **.astro** ファイルを使いますが、その中で React, Vue, Svelte などのコンポーネントをそのまま使うことができます。<br>

私は Web 開発の勉強として React (Next.js) にも興味がありますが、Astro なら「基本は Astro で書きつつ、一部の複雑なUIだけ React で書く」という使い分けができるのが魅力的でした。

### まとめ：エンジニアのポートフォリオに最適
- 速い: 不要なJSがないので読み込みが一瞬。
- 書きやすい: JSX に似た記法で直感的に書ける。Markdown との相性が抜群。
- 拡張性: 必要になれば React などを混ぜられる。

ブログや研究成果のアウトプット用として、現時点での「最適解」の一つだと感じています。
